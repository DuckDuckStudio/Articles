# 在提 PR 前，请问自己几个问题

1. HEAD 和 Base 分支正确吗？
   1. 是否将 PR 直接提到了上游仓库？
   2. 是否将 PR 直接跳过 develop 分支提交到 main？
2. PR 有明确链接议题或问题吗？
3. 在本地对 PR 中的修改做过测试吗？
4. 对这些修改编写了测试了吗？
   - 包括像测试工作流中的步骤，tests 等
5. 这是一个针对新功能的 PR 吗？
   - 如果是，请先提出议题讨论
6. 我的分支是最新的吗？
   - 在提 PR 前更新分支以避免合并冲突
7. 我的 PR 通过现有测试了吗？
8. 项目进一年内有新提交吗？
   - 如果没有，则代表该项目可能已经被“遗弃”
